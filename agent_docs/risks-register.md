# Сводный реестр рисков ESD ServiceDesk

## Введение

Реестр консолидирует все ключевые риски проекта Оператора «Единой Среды Доступа» (ЕСД) — технические, HR, коммерческие, регуляторные, репутационные и финансовые. Назначение:

- единый источник для управления рисками на уровне C-level и Совета директоров;
- основа для quarterly review (рекомендация — ежеквартальный обзор силами **CISO + HR + CFO** с приглашением CEO/CTO/COO);
- привязка каждого риска к ответственному из RACI (хранится в `NDA/team-plan.md` вне репозитория);
- определение trip-wires и ранних индикаторов для автоматического срабатывания митигаций.

> **Важно:** конкретные финансовые показатели, тарифы и суммы инвестиций — в NDA-материалах (`NDA/financial-model.md`, `NDA/investment-plan.md`, `NDA/tariff-model.md`). Этот документ описывает риски без раскрытия финансовых деталей и доступен разработчикам.

**Регламент обновления:**
- Quarterly review — раз в 3 мес.
- Внеплановое обновление — при срабатывании trip-wire или при появлении нового риска (например, изменение регулирования, отказ ключевого партнёра).
- При материализации риска — фиксация инцидента в постмортеме, обновление вероятности и митигации.

Связанные документы: [regulatory-roadmap.md](regulatory-roadmap.md), [adr.md](adr.md). План команды и RACI — `NDA/team-plan.md`.

## Раздел 1. Категории рисков

| Категория | Описание | Основной владелец |
|---|---|---|
| **Технические** | Отказы инфраструктуры, узкие места производительности, проблемы интеграции с внешними API, утечки ПДн через технические уязвимости. | CTO / CISO |
| **HR** | Выгорание core-team, дефицит сениоров с нужным опытом (госконтур + high-load), уход ключевых людей до vesting-cliff. | CEO / HR Lead |
| **Коммерческие** | Сопротивление УК, конкуренты, отказ B2B-партнёров, низкая конверсия ОСС, недостижение KPI продаж. | CEO / CRO / Head of Sales |
| **Регуляторные** | Закрытие/изменение партнёрского API ЕПГУ, отказ в аттестации ФСТЭК, срыв сертификации ПАК, изменения в 152-ФЗ или ЖК РФ. | GR / CISO |
| **Репутационные** | Инциденты с участием курьеров-злоумышленников, утечки данных, негативные публикации, политические скандалы. | CEO / Маркетинг |
| **Финансовые** | Кассовый разрыв, перерасход бюджета, недостижение MRR, проблемы с привлечением bridge-round. | CFO / CEO |

## Раздел 3. Trip-wires и ранние индикаторы

Автоматические сигналы, при срабатывании которых запускается заранее определённое действие. Snapshot — ежемесячный, с эскалацией на C-level через 24 ч после срабатывания.

| Что мониторить | Порог | Действие | Триггер связан с риском |
|---|---|---|---|
| Cash runway (CFO dashboard) | < 6 мес runway при текущем burn rate | Автоматический откат scaling-плана: заморозка найма, сокращение маркетинга на 50 %, активация bridge-round в течение 30 дней | R16, R17 |
| MRR (CFO dashboard) | ниже планового порога к мес. 9 (конкретные значения — `NDA/financial-model.md`) | Пересмотр go-to-market стратегии: внеплановый strategy review CEO + CRO + CPO + Совет директоров | R11, R13, R14, R16 |
| eNPS (HR survey) | < 30 при quarterly опросе | Focus на удержание: 1-on-1 с каждым Senior+; пересмотр компенсационного пакета; внеплановый ESOP-grant top-performers | R08, R09 |
| Аварии BLE (alerts по `device-gateway`) | > 1 % неуспешных handshake в неделю | Пауза rollout новых объектов; root cause analysis; патч прошивки ПАК в течение 7 дней | R19 |
| Отказ в ФСТЭК-аттестации | официальное письмо ФСТЭК с замечаниями | План B: поэтапный rollout только в не-чувствительных контурах (без ПДн жителей); инициация re-audit через 90 дней | R03 |
| Задержка партнёрского API ЕПГУ | > 3 мес от планового срока (target — мес. 10) | Негоциация альтернативных каналов: white-label, региональные порталы «Умный город»; внеплановый strategy review с участием Совета директоров | R01, R05, R07 |
| ACL p99 latency | > 300 ms (BLE-критичный порог, пользователь стоит у двери) | Critical alert → on-call SRE; немедленный rollback последнего деплоя; инцидент с приоритетом P1 | R19 |
| Утечка ПДн (DLP / SIEM) | любое срабатывание правил DLP на эксфильтрацию | Немедленное отключение учётной записи; форензик-анализ; уведомление РКН в течение 72 ч (152-ФЗ); PR-протокол | R06, R23 |
| Конверсия ОСС | < 30 % успешно проведённых ОСС от инициированных | Пересмотр шаблонов решений; усиление методической поддержки УК; financial incentive для собственников | R13 |
| Уход ключевого сотрудника (C-level / Lead) | заявление об уходе | Обязательный exit-interview с CEO; немедленный найм-replacement; knowledge transfer 30 дней; пересмотр RACI | R08, R10 |
| Cost overrun (CFO monthly review) | > 10 % от plan на месяц | Внеплановый CFO + CEO review причин; corrective action plan в течение 7 дней | R17 |
| Heartbeat ПАК-устройств | > 5 % устройств offline > 1 ч | Auto-escalation L2; выезд инженера; review device-gateway capacity | R19 |
| Conversion подключения УК | < 50 % от заявок до активного объекта (target ≥ 70 %) | Пересмотр onboarding-плейбука; усиление CSM-команды; root cause по этапам воронки | R11 |
| Анонсы конкурентов о partner API с Госуслуги.Дом | любой публичный анонс A-Access / Simplgate / VisitorControl | Внеплановый strategy review CEO + CRO; ускорение Term Sheet с ПАО МРТС; усиление эксклюзивности с производителями домофонов | R12 |

## Раздел 4. План эскалации

Чёткая схема, кто принимает решение по риску в зависимости от уровня влияния.

| Уровень | Кто принимает решение | Когда эскалируется | SLA от триггера до решения |
|---|---|---|---|
| **Уровень 1: Head of department** | Head of Sales / CSM / Engineering / SRE / etc. | Срабатывание trip-wire с локальным влиянием (например, конверсия подключения УК ниже плана; отдельный инцидент BLE) | **24 часа** на принятие решения и план действий |
| **Уровень 2: C-level** | CEO / CTO / CFO / COO / CISO / CRO | Системный риск (cash runway, MRR < target, утечка ПДн, отказ в ФСТЭК); влияние на стратегию | **7 дней** на решение, включая внеплановый strategy review |
| **Уровень 3: Совет директоров / Инвесторы** | Совет директоров (CEO + 2 фаундера + 1 представитель инвестора) | Экзистенциальный риск (R01 — отказ ПАО МРТС; R10 — уход фаундера; R16 — кассовый разрыв при невозможности bridge-round); пересмотр ADR | **30 дней** на решение, оформляется как новый ADR |

**Дополнительные правила:**
- **Эскалация на правоохранительные органы / РКН / ФСБ** — обязательна в течение 72 ч при материализации R06 (утечка ПДн), R15 (инцидент с курьером-злоумышленником), R23 (инсайдерская атака). Ответственный: Юрист + CISO.
- **Эскалация в страховую** — при материализации R06, R15 — в течение 24 ч после фиксации инцидента.
- **PR-протокол активируется** при R06, R15, R23, R10 (если уход фаундера публичный). Ответственный: Маркетинг + CEO.
- **Регулярный ежеквартальный risk review** — обязательное участие CEO, CFO, CISO, HR Lead. Все остальные C-level — по приглашению. Результат — обновлённый snapshot реестра + список triggered trip-wires + план корректирующих действий.
