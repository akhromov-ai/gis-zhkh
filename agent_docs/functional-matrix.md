# Функциональные матрицы ESD ServiceDesk

> Две матрицы: (1) распределение прав ролей по модулям системы (CRUDX) и (2) календарь раскатки модулей по этапам roadmap. Список ролей — 12 единиц по ADR-006 (только роли, относящиеся к разработке и эксплуатации продукта). Список модулей — 14 укрупнённых функциональных областей (детальная разбивка на микросервисы — в `architecture.md` и `functional-spec.md`).

### 1.1. Пояснения по областям ответственности

- **Житель** работает в Госуслуги.Дом (BFF Mobile) — только в своём контексте.
- **УК-админ** — единственная роль с правом CRUD на объекты, помещения, жителей, ОСС в рамках своей УК.
- **Диспетчер** работает в смене с конкретным набором объектов; права ограничены по `object_ids` в JWT-claims.
- **Охранник** — read-only доступ к журналу проходов своей точки; для экстренной ручной выдачи использует Service Desk.
- **Монтажник** видит только свои задания и свой профиль на бирже.
- **B2B-партнёр** работает исключительно через Public B2B Gateway, не имеет UI-доступа.
- **Гость** — нет учётной записи, действует по deep-link от жителя.
- **L1** обрабатывает первичные обращения; **L2** — эскалации, имеет доступ к редактированию тикетов и проверке логов.
- **ИБ** — read-only по всем системам, CRUD по audit-log; реагирует на алерты SIEM.
- **ИТ-админ** — управление инфраструктурой, секретами, конфигами; не видит бизнес-данных.
- **Комплаенс-аудитор (Минцифры / РКН)** — read-only с правом экспорта по аудиту, журналу проходов, согласиям ОПД.

> Модуль **«Биллинг и квоты»** относится к управлению лимитами и состоянием подписки УК (active / grace / suspended / terminated, счётчики использования, апгрейды по достижении порогов). Конкретные тарифные планы и параметры лимитов настраиваются через конфиг и находятся вне технического контура этой документации.

### 2.1. Критические зависимости версий

- **Кабинет жителя (mobile) v1.0** требует подписания партнёрского соглашения с Оператором Госуслуги.Дом (см. `regulatory-roadmap.md`). Это критический путь по ADR-004.
- **ACL realtime v1.0** требует готовности Redis Cluster и device-gateway на Go.
- **ОСС-конструктор v2.0** требует аккредитации ИС в ГИС ЖКХ как «Commercial IS» (для доступа к закрытому реестру с площадями помещений).
- **Партнёрский шлюз v2.0** требует завершения интеграционных треков с каждым из B2B-партнёров (Yandex / OZON / WB / СДЭК).
- **Биллинг v2.0** требует интеграции с эквайрингом (Тинькофф/Яндекс.Касса) и Контур.Диадок.
- **ИБ-аудит v1.1 (ФСТЭК-21)** — обязательное условие выхода в коммерческие УК. Аттестация занимает 8–14 мес.
